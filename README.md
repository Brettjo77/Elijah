# Elijah Ford Microsoft Experience Website

This is a static website prepared for GitHub Pages. It is designed to be edited in VS Code with GitHub Copilot and Claude AI, then published through GitHub without needing a build step.

## What is in this project

- `index.html`: the website content and page structure.
- `styles.css`: the visual design.
- `design-pack/`: place Claude's design pack here when it is ready.
- `assets/images/`: place photos, screenshots, and approved graphics here.
- `CLAUDE.md`: guidance Claude can read before editing the site.

## GitHub setup

1. Create a new public GitHub repository.
2. Suggested repository name: `elijah-ford-microsoft`.
3. In VS Code, open the Source Control panel.
4. Initialize the repository if VS Code asks.
5. Commit the starter files with a message like `Create starter site`.
6. Publish or push the repository to GitHub.
7. In GitHub, open the repository settings.
8. Go to **Pages**.
9. Under **Build and deployment**, choose **Deploy from a branch**.
10. Select the `main` branch and `/root` folder.
11. Save, then wait for GitHub to provide the public Pages URL.

## Editing workflow

Use plain language prompts like these:

> Update the homepage using the design pack in `design-pack/`. Keep it static HTML and CSS for GitHub Pages.

> Replace the placeholder Microsoft experience text with this story, and keep the tone polished and personal.

> Add the photos for the page and make sure the site still works on mobile.
