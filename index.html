My day at Microsoft

I arrived at the Microsoft office at 9 o'clock and went straight into a team meeting. This was the monthly all hands, and I was welcomed very warmly by the EMEA GBBs. It was a good way to start the day and see how the wider team comes together.

After the meeting we went on a tour around the Microsoft campus. I saw a lot of the different working areas and got a feel for what the office is like to work in, although it was very quiet that day.

We then sat down to look at some of the Copilot tools that are available. We tried it inside PowerPoint and Word, and after a bit of prompting it produced some Warhammer themed examples, which was a good way to see how the AI responds to what you ask it.

Next we visited Gareth, an old school friend of Brett's who works within Xbox Studios. He gave us a private tour of the Xbox area, where we spent time understanding the games that are made there and how they are developed. We also went into the server room, which was full of PlayStation and Xbox developer kits running AI to test all of the games.

We then joined a meeting with a customer. The conversation was about how Copilot searches the internet, and we talked through some of the technical capabilities behind it.

After lunch we built this website, which is described below.

How we built this website

We built this whole site without writing any code by hand. The idea was simple. Rather than learning to code, you describe what you want in plain words and work with the AI to make it real.

First we used Claude Design to create the look of the site. We described what we wanted out loud using voice transcription, and asked for three design options to choose from. The brief was a simple, professional template with a sidebar for Home, Pictures and Report, neutral blue and white colours, and smooth animations between pages. We chose a modern editorial magazine style with a smooth slide and fade transition, and picked the third design option.

Once we were happy with the design, we downloaded the design pack and saved it into a project folder alongside the five photos from the day. We opened that folder in VS Code and used Claude Code to turn the template and the real photos into the finished site. It recognised the photos on its own and used them as proper images rather than placeholders.

To put the site online we created a repository on GitHub, which is a free place for a website to live. We connected VS Code to that repository, published the files, and switched on GitHub Pages by choosing to deploy from the main branch. That gave us a live web address anyone can visit.

The tools we used were Claude Design and Claude Code for the build, Whisper for the voice transcription, VS Code as the editor, and GitHub for hosting. The main thing to take away is that you do not have to know how to do all of this in advance. You explain what is in your head, the AI helps you build it, and you learn as you go.
